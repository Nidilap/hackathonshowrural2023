# Documentação

    O presente documento tem o intuito de descrever as convenções de escrita, bibliotecas utilizadas, dentro outros recursos
empregados nesse projeto WEB.

# Convenções de escrita

    As convenções são muito importantes para manter uma escrita padrão em todo o código e facilitar as futuras manutenções. No geral,
os códigos foram escritos todos em português.

    * Java

        - Padrão "camelCase"
        - Use "is" no começo de variáveis/métodos booleanos

    * Java (API)

        - Padrão "kebab-case"

    * Java (POM)

        - Primefaces: provido pelo AdminFaces | v10.0.0
        - FontAwesome: versão 5.15.4 definida no Template (os ícones do Primefaces devem estar como TRUE no web-xml)

    * HTML

        - Padrão "camelCase" para nomeação de IDs
        - Coloque uma abreviação do tipo de componente ao final do nome do ID

    * CSS

        - Padrão "kebab-case"

    * JS

        - Padrão "camelCase"
